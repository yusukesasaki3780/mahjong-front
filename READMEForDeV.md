# 新規ページ作成時のファイル配置ガイド

プロジェクト内で新しい画面（ページ）を追加する際は、以下のディレクトリポリシーに沿ってファイルを配置してください。

## 1. 画面コンポーネントの配置
- ページ本体は `src/views/<ドメイン>/` 配下に `.vue` ファイルとして作成します。
  - 例：プロフィール関連 → `src/views/profile/`、シフト関連 → `src/views/shifts/`
  - 汎用的なページでドメインが明確でない場合は、役割に応じた新規フォルダを `views` 直下に切ってください。

## 2. ルーティング設定
- ルート定義は `src/router/` 配下のルート設定ファイルを編集します。
  - 既存の `router/index.ts` などにページコンポーネントをインポートしてパスを追加してください。

## 3. API クライアント
- 新しいページで独自の API を呼ぶ場合、`src/api/` 配下にドメイン別の `.ts` ファイルを追加・拡張します。
  - 既存ファイルで対応できる場合は追記し、役割が重複する場合は既存ファイルを流用してください。

## 4. 共通コンポーネント
- 共通で再利用できる UI は `src/components/` 配下に配置します。
  - ページ固有でない汎用コンポーネントは `components/common/`
  - 特定機能領域に紐付く場合は `components/<ドメイン>/`

## 5. 型定義・ユーティリティ
- ページ専用の型は `src/types/` に、ユーティリティ関数は `src/utils/` もしくは該当ドメインの `composables/` などへ配置してください。

## 6. スタイル
- ページ固有のスタイルは原則その `.vue` ファイル内の `<style scoped>` に記述します。
- 共有スタイルを追加する場合は `src/styles/` を利用してください。

## 7. テスト
- 単体テストが必要なロジックを追加する場合は、対応する `tests/` または `src/__tests__/` に Spec を作成します。

これらを満たすことで、ディレクトリ構成の一貫性を維持しながら新規ページを実装できます。 README 本体にも必要に応じて利用方法を追記してください。
